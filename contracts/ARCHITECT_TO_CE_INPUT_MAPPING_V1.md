# Architect to CE Input Mapping v1

Status: contract_definition_v1_hardened  
Input schema: `ev4-architect-output-contract@1.0.0`  
CE ingestion schema: `ev4-architect-to-ce-input-package@1.0.0`  
Owning repo: `rezahh107/EV4-Constructability-Engineer-Repo`

## Purpose

This contract defines how normalized Architect output becomes a strict CE input package.

It prevents CE from receiving conceptual prose and having to guess executable structure. It also prevents Architect output from being mistaken for Builder-ready execution data.

## Boundary

Accepted input to CE must remain non-executable:

```yaml
accepted_for_ce_review: true
builder_executable_allowed: false
builder_ready_status: not_builder_ready_without_ce_proof
production_ready_allowed: false
```

CE may produce a Builder Executable Package only after constructability review proves all required strategy details and Builder has zero remaining strategy decisions.

## Required schema

Use:

```text
schemas/architect_ce_input_package.v1.schema.json
```

The schema is strict:

- `additionalProperties: false`
- all review units are explicit
- all proof states are explicit
- unmapped fields are forbidden
- semantic guessing is forbidden
- Builder package emission is forbidden at ingestion time
- `forbidden_work` must survive Architect → CE conversion
- `source_payload_ledger` must survive Architect → CE conversion
- `evidence_gaps_for_ce` must survive Architect → CE conversion
- cross-reference consistency checks are required before CE ingestion
- proof-missing default is fail-closed: `blocked`

## Mapping rules Architect → CE

| Source | Target | Mapping rule | Failure condition |
|---|---|---|---|
| `schema` | `source_architect_schema` | require `ev4-architect-output-contract@1.0.0` | reject other schemas |
| `selected_candidate_id` | `selected_candidate_id` and `architect_contract.selected_candidate_id` | copy exact | reject if changed, blank, or malformed |
| `selected_candidate_locked` | `selected_candidate_locked` | require `true` | reject if false/missing |
| `source_payload_ledger[]` | `source_payload_ledger[]` | copy exact | reject if missing or lossy |
| `approved_structure_tree[]` | `approved_structure_tree[]` | copy normalized nodes | reject if node IDs are missing or malformed |
| `approved_class_names[]` | `approved_class_names[]` and `architect_contract.approved_class_names[]` | copy exact unique list | reject duplicates, blanks, or unresolved names |
| `ce_review_units[]` | `ce_review_units[]` | copy exact | reject if any executable node has no review unit |
| `evidence_gaps_for_ce[]` | `evidence_gaps_for_ce[]` and `ce_review_units[].interrogation_inputs.*.state` | preserve and convert to `not_proven` or `blocked` | reject if proof is missing but not represented |
| `forbidden_work[]` | `forbidden_work[]` and `architect_contract.forbidden_work[]` | copy exact as negative constraints | reject if missing or reduced |
| `responsive_qa_seed` | `ce_review_units[].interrogation_inputs.responsive` | map without inference | reject if responsive behavior is implied but absent |
| Builder-ready fields | none | forbidden | reject if present |
| Unmapped freeform fields | none | forbidden | reject if present |

## Proof-state mapping

CE ingestion must use explicit proof states:

```text
not_required
evidence_backed
not_proven
blocked
```

Rules:

- `required: false` forces `state: not_required` and empty `evidence_refs`.
- `state: evidence_backed` requires at least one `evidence_refs[]` item.
- missing evidence is never treated as false and never upgraded to proven.
- if proof is required but absent, the ingestion default is `blocked`.

Boundary states must use:

```text
preserved
not_proven
requires_architect_amendment
blocked
```

If `crosses_boundary: true`, CE input must use either:

```text
requires_architect_amendment
blocked
```

## Required identity and trace checks

A CE input package must include these checks as true:

```yaml
identity_consistency_checks:
  selected_candidate_id_matches_architect_contract: true
  approved_class_names_match_architect_contract: true
  all_review_units_target_existing_nodes: true
  all_evidence_gaps_target_existing_units: true
  forbidden_work_preserved_from_architect_output: true
```

These checks are not a license for CE to guess. They are a pre-ingestion assertion that must be generated by the Gate or adapter before CE review starts.

## Pre-ingestion validation checklist

A CE input package is acceptable only when all checks below are true:

```yaml
verified_by: project_gate_validator | ce_adapter_validator | manual_contract_review
architect_schema_valid: true
no_additional_properties: true
no_builder_ready_claim: true
selected_candidate_locked: true
approved_class_names_resolved: true
all_review_units_mapped: true
source_payload_ledger_preserved: true
forbidden_work_preserved: true
evidence_gaps_preserved: true
identity_consistency_verified: true
unmapped_fields: []
```

## CE no-guessing rule

CE must not infer or invent:

```text
geometry
source_target_anchors
asset source
placeholder policy
overlay strategy
containment
z-index
responsive behavior
interaction
Dynamic Loop
accessibility evidence
Elementor UI control path
class boundary changes
structure boundary changes
```

If any of these are required but not proven at ingestion, CE input starts from:

```text
blocked
```

CE may later classify the review outcome as `needs_user_evidence` or `needs_architect_amendment`, but the ingestion package itself must fail closed.

## Builder package gate

At ingestion time:

```yaml
builder_package_emission_allowed_at_ingestion: false
semantic_guessing_allowed: false
ambiguous_fields_allowed: false
default_status_when_proof_missing: blocked
```

A Builder Executable Package may appear only after CE review satisfies the existing package-mode requirements: `constructability_status: executable_ready`, `builder_decisions_required: 0`, and `blocking_dependencies: []`.

## Critic hardening decisions applied

The v1 mapping was tightened after review in four ways:

1. `forbidden_work` is now required in CE input and inside `architect_contract`.
2. `source_payload_ledger` is now required in CE input to prevent evidence provenance loss.
3. `evidence_gaps_for_ce` is now required in CE input to prevent missing proof from disappearing during conversion.
4. proof-missing default is now `blocked`, not a selectable status.
