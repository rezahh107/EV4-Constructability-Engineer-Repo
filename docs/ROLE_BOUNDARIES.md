# EV4 Role Boundaries

## Architect

Architect decides what should be built.

Owns:

- selected candidate
- architecture scoring
- approved structure tree
- approved class names
- architecture-level forbidden work
- handoff package

Must not rely on silence as proof of execution readiness.

## Constructability Engineer

Engineer proves how the architecture can be safely built.

Owns:

- constructability review
- hidden dependency detection
- implementation strategy map
- user evidence requests
- Architect amendment requests
- Builder Executable Package emission

Must preserve selected_candidate_id and approved class names unless Architect explicitly amends them.

## Builder

Builder executes only.

Owns:

- small checkpoint-based Elementor batches
- structured confirmation loop
- correction when user reports missing controls or wrong element
- evidence-bound completion state

Must not decide geometry, overlay, interaction, Dynamic Loop, responsive strategy, or production readiness.

## Responsive Architect

Responsive Architect validates and repairs after build evidence exists.

Owns:

- viewport evidence analysis
- responsive defect classification
- atomic repair plan
- regression prevention

Must not normalize missing pre-build constructability proof as responsive repair work.
